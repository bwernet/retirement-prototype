# the bow — Agentic Wedding-Marketplace Concept Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build `the-bow.html` — a self-contained, full-viewport, scripted "semi-working" prototype of an agent-first wedding-planning marketplace ("the bow"), rebuilt from Beth's 9 prototype screens, for the Experiments section of her portfolio.

**Architecture:** All testable substance lives in `bow-model.js` (pure data + functions: `BRAND` tokens, `VENUES`, `SCRIPT` beat graph, budget arithmetic, input matcher — unit-tested with `node:test`). `the-bow.html` holds markup + CSS + an inline module script (the render engine) that imports the model. `build.mjs` inlines fonts + model into `dist/the-bow.html`. Static-first where possible: the home dashboard renders complete from markup; the script layers on the storyline engine.

**Tech Stack:** Vanilla HTML/CSS/JS, Node ≥18, no dependencies, no package.json. New `bow-fonts.css` (Figtree, base64) generated by a new `build-bow-fonts.mjs` mirroring `build-babytalk-fonts.mjs`.

**Spec:** `docs/superpowers/specs/2026-07-16-the-bow-agentic-concept-design.md` — read it before starting any task. The canonical budget table there is law.

## Global Constraints

- **Whitelabel is absolute:** the strings `Knot`, `knot`, `theknot` must NOT appear anywhere in `bow-model.js`, `the-bow.html`, or `dist/the-bow.html` (tests enforce; note `.bow-knot` class names would false-positive — don't use "knot" in any identifier). Brand name is lowercase **"the bow"** everywhere.
- **Canonical numbers** (never re-derive by hand; tests assert): goal `40000` → `45000` after increase; venue package `7200 + 18000 + 2100 = 27300`; pre-booking committed `17500 = 1700 paid + 15800 booked` (photographer 6000 + videographer 5000 + florist 3300 + DJ 1500); planned remaining `6500` (dress 3000, tux 1000, cake 1000, hair & makeup 900, stationery 600); projected after add `51300`; over-goal `11300`; rebalance trims `−6900` → `44400` (catering 18000→12500, fees 2100→1550, dress→2500, stationery→350, cake→900); goal-increase result "`$600` under"; deposit `5600`; post-booking spent `23100`; negotiation: contract `7450`, honored `7200`. "It's fine, proceed" branch books at `7450` (model-computed totals shift by +250).
- **Verbatim copy pins:** every line marked `PIN` in Task 2/3 SCRIPT data ships exactly as written (they come from Beth's screens) and is asserted by tests.
- **Couple:** Tony & Carmella. Venues: Willow Shore Lodge, Harborcrest Pavilion, Silver Pines Retreat, Blue Horizon Club, Evergreen Point Estate (Evergreen never replies). Coordinator: Maya (Willow Shore). No other personal names anywhere, including commits.
- **Header chips:** 📅 9/20/26 · 👥 125 guests · 📍 Columbus, OH · 🐷 $17.5k spent | $40k budget. Spent chip changes ONLY at booking (→ $23.1k). Budget chip changes only if goal increased (→ $45k).
- **Emoji are allowed** — this product's own voice uses them (🐷 ✅ 🚫 ⚠️ 🎉 etc.), matching the screens.
- **Motion:** restrained; text streaming, panel slides, one count-up per act, single-play interstitials; nothing loops. `prefers-reduced-motion` → instant everything.
- **Accessibility:** chips/cards/panels tabbable in visual order; visible focus (gradient ring); panels trap + restore focus; agent messages announced via `aria-live="polite"` once complete; CTA pink must pass AA for its text size or be darkened stepwise (record shipped value in `BRAND`).
- Build stays dependency-free; `dist/the-bow.html` must contain **zero external references** (no `http` fetches at runtime, no `<link href=`, no `src="http`).
- Full-viewport app: no postMessage height reporting.
- Work on branch `the-bow` off `main`.

```js
export function reachableBeats(fromId) {
  const seen = new Set([fromId]);
  const queue = [fromId];
  while (queue.length) {
    const beat = SCRIPT[queue.shift()];
    for (const c of beat?.chips ?? []) if (!seen.has(c.goto)) { seen.add(c.goto); queue.push(c.goto); }

  }
  return [...seen];
```

```js
export function matchInput(text, currentBeatId) {
  const words = text.toLowerCase().replace(/[^a-z0-9\s-]/g, '').split(/\s+/).filter(w => w.length > 2);
  if (!words.length) return null;
  // Direct chip targets only: the whole graph is reachable from home, so a
  // wider net would let "book willow shore lodge" skip the entire story.
  const candidates = [...new Set((SCRIPT[currentBeatId]?.chips ?? []).map(c => c.goto))]
    .filter(id => id !== currentBeatId && id !== 'fallback');
  let best = null, bestScore = 0;
  for (const id of candidates) {
    const keys = (SCRIPT[id].match ?? []).join(' ').toLowerCase();
    let score = 0;
    for (const w of words) if (keys.includes(w)) score++;
    // phrase bonus: multi-word match keys that appear whole in the input
    for (const k of SCRIPT[id].match ?? []) {
      if (k.includes(' ') && text.toLowerCase().includes(k)) score += 2;
    }
    if (score > bestScore) { bestScore = score; best = id; }
  }
  return bestScore >= 1 ? best : null;
}
```

Engine behavior (implement exactly):

1. `go(id)`: look up beat. If beat has `thread` → show `#thread`, set title; append user bubble (if any) instantly; show typing indicator (three-dot, 600ms); stream agent blocks sequentially. 4. Typed input (`#ask`, `#ask-thread`): Enter → `matchInput(text, state.beat)`; hit → `go(match)` **with the typed text as the user bubble instead of the beat's scripted one**; null → append user bubble + fallback beat's text + re-render current beat's chips.

7. `#back` → home (state preserved). `#restart` → full reset: rebuild state, `data-state="fresh"`, clear transcripts, `go('home')` view.
8. Fallback beat never changes `state.beat`.
